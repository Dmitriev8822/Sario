# Sario

Sario — лёгкая браузерная пиксельная мини-игра-подарок, вдохновлённая классическими платформерами.
Проект написан на чистых HTML, CSS и JavaScript, поэтому его можно открыть локально или разместить на обычном статическом хостинге.

## Запуск

Откройте `index.html` в браузере или запустите любой локальный статический сервер:

```bash
python3 -m http.server 8080
```

После этого перейдите на `http://localhost:8080`.

## Структура

- `index.html` — основной HTML-каркас, главный экран, экран игры, topbar и панель редактора уровня.
- `style.css` — общий пиксельный стиль, адаптивность, canvas, topbar и элементы редактора.
- `game.js` — движение персонажа, коллизии, загрузка JSON-уровней, переходы между локациями, редактор блоков/предметов, сохранение уровня и ачивки.
- `assets/levels/` — JSON-файлы уровней и `manifest.json` со списком локаций.
- `assets/player/<costume>/` — папки костюмов персонажа, каждая содержит `idle.png`, `jump.png`, `run1.png`, `run2.png`, `run3.png`.
- `assets/attributes/` — картинки предметов и `manifest.json` с данными ачивок: именем файла, названием и описанием.

## Настройка уровней

Стартовые данные локаций хранятся в `assets/levels/`. Файл `assets/levels/manifest.json` задаёт порядок загрузки:

```json
{
  "levels": ["location-1.json", "location-2.json"]
}
```

Каждый JSON уровня описывает фон, ширину мира, старт игрока, блоки, предметы и чекпоинты костюмов:

```json
{
  "id": "location-1",
  "title": "Первая локация",
  "background": "assets/background/1.png",
  "worldWidth": 6623,
  "start": { "x": 80, "y": 272 },
  "blocks": [
    { "x": 360, "y": 286, "w": 124, "h": 18 }
  ],
  "items": [
    {
      "id": "military-ticket",
      "x": 6460,
      "y": 186,
      "w": 42,
      "h": 42,
      "src": "assets/attributes/Military ticket.png",
      "title": "Военный билет",
      "text": "Ты получил военный билет — впереди новая локация.",
      "transitionText": "Документ на руках. Открывается следующий этап пути!",
      "nextLevel": "location-2"
    }
  ],
  "costumeCheckpoints": [],
  "events": []
}
```

- `background` — фон конкретной локации из `assets/background/`.
- `worldWidth` — ширина мира для камеры и пола. Автоматического финиша по координате больше нет: переход между уровнями запускают только предметы с `nextLevel`.
- `worldWidth` — ширина мира для камеры, пола и финишной зоны.
- `start` — стартовая позиция игрока при входе на уровень.
- `blocks` — платформы уровня. `x` и `y` задают позицию, `w` и `h` — размер.
- `items` — вручную размещённые предметы. Если у предмета есть `nextLevel`, после подбора запускается плавный переход на указанный уровень.
- `transitionText` — текст карточки, который показывается перед переходом.
- `costumeCheckpoints` — вертикальные чекпоинты смены костюма на всю высоту игрового поля. `x` задаёт позицию чекпоинта, `costume` — имя папки костюма в `assets/player/`.
- `events` — события текущей локации, если нужно показывать карточки около важных точек.

## Автообновление предметов

Чтобы добавить новый собираемый предмет, положите изображение в `assets/attributes/` и опишите его в `assets/attributes/manifest.json`. Поддерживаются `.png`, `.jpg`, `.jpeg`, `.webp`, `.gif` и `.svg`. Игра сначала читает manifest как основной источник данных. Если manifest недоступен, она пробует листинг папки и генерирует название и описание из имени файла.

Пример `assets/attributes/manifest.json`:

```json
{
  "attributes": [
    {
      "file": "Military ticket.png",
      "title": "Военный билет",
      "text": "Ты открыл ачивку «Военный билет» — важный этап пройден."
    }
  ]
}
```

- `file` — имя файла картинки в `assets/attributes/` или полный путь к файлу.
- `title` — название ачивки, которое показывается в игре и редакторе.
- `text` — описание ачивки в карточке при подборе предмета.
- `autoPlace: false` — отключает автоматическое размещение предмета на платформах, чтобы разместить его вручную в JSON уровня.

Для совместимости manifest всё ещё может быть массивом строк или содержать старое поле `files`, но в таком формате название и описание будут сгенерированы из имени файла.

## Автообновление костюмов

Чтобы добавить новый костюм игрока, создайте папку в `assets/player/`, например `assets/player/winter/`. Внутри должны быть пять PNG-спрайтов с фиксированными именами:

- `idle.png`
- `jump.png`
- `run1.png`
- `run2.png`
- `run3.png`

При каждом запуске игра сначала читает `assets/player/manifest.json`, а если файл недоступен — пробует листинг `assets/player/`. Такой manifest надёжнее на хостингах, где directory listing отключён или кэшируется. Имя `id` используется как идентификатор костюма в экспортируемом JSON уровня.

Пример `assets/player/manifest.json`:

```json
{
  "costumes": [
    { "id": "young", "title": "Young", "path": "assets/player/young" },
    { "id": "army", "title": "Army", "path": "assets/player/army" }
  ]
}
```

Если для костюма нужны нестандартные пути к отдельным позам, можно добавить объект `sprites` с ключами `idle`, `jump`, `run1`, `run2`, `run3`.

## Редактор уровня в игре

1. Нажмите `Начать игру`.
2. Нажмите кнопку `Редактор` в верхней панели.
3. Выберите инструмент `Блок`, `Предмет` или `Костюм`.
4. Если выбран `Предмет`, во втором списке выберите конкретную картинку из `assets/attributes/` — например `Laptop` или `Military ticket`.
5. Если выбран `Костюм`, выберите костюм из списка папок `assets/player/`. Клик добавит вертикальный чекпоинт на всю высоту игрового поля: когда игрок дойдёт до него, спрайты игрока сменятся на выбранный костюм.
6. Кликните по canvas, чтобы добавить объект.
7. Используйте `Alt + клик`, `Shift + клик` или правую кнопку мыши, чтобы удалить объект выбранного типа.
8. Нажмите `Экспорт`, чтобы скопировать JSON всех уровней с блоками, предметами и чекпоинтами костюмов и при необходимости перенести изменения в файлы `assets/levels/*.json`.
9. Нажмите `Сброс`, чтобы удалить сохранённый в браузере вариант и вернуться к дефолтному уровню.

Изменения редактора автоматически сохраняются в `localStorage` браузера под ключом `sario.levels`.
