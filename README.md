# Sario

Sario — лёгкая браузерная пиксельная мини-игра-подарок, вдохновлённая классическими платформерами.
Проект написан на чистых HTML, CSS и JavaScript, поэтому его можно открыть локально или разместить на обычном статическом хостинге.

## Запуск

Откройте `index.html` в браузере или запустите любой локальный статический сервер:

```bash
python3 -m http.server 8080
```

После этого перейдите на `http://localhost:8080`.

## Структура

- `index.html` — основной HTML-каркас, главный экран, экран игры, topbar и панель редактора уровня.
- `style.css` — общий пиксельный стиль, адаптивность, canvas, topbar и элементы редактора.
- `game.js` — конфигурация уровня, движение персонажа, коллизии, редактор блоков/предметов, сохранение уровня и ачивки.
- `assets/player/<costume>/` — папки костюмов персонажа, каждая содержит `idle.png`, `jump.png`, `run1.png`, `run2.png`, `run3.png`.
- `assets/attributes/` — PNG-картинки предметов, которые можно разместить на уровне и показать при сборе.

## Настройка уровня

Все стартовые данные уровня лежат в `CONFIG.level` внутри `game.js`:

```js
level: {
  blocks: [
    { x: 360, y: 286, w: 124, h: 18 },
  ],
  items: [
    {
      x: 760,
      y: 202,
      w: 34,
      h: 34,
      src: "assets/attributes/example.png",
      title: "Новая ачивка",
      text: "Ты нашёл предмет уровня!",
    },
  ],
  costumeCheckpoints: [
    { x: 1600, costume: "young" },
  ],
}
```

- `blocks` — платформы уровня. `x` и `y` задают позицию, `w` и `h` — размер.
- `items` — дополнительные вручную размещённые предметы. Базовый набор теперь создаётся автоматически из картинок в `assets/attributes/`: игра пробует прочитать список файлов из папки, затем `assets/attributes/manifest.json`, а названия ачивок берёт из имён файлов.
- `costumeCheckpoints` — вертикальные чекпоинты смены костюма на всю высоту игрового поля. `x` задаёт позицию чекпоинта, `costume` — имя папки костюма в `assets/player/`.

## Автообновление предметов

Чтобы добавить новый собираемый предмет, положите изображение в `assets/attributes/`. Поддерживаются `.png`, `.jpg`, `.jpeg`, `.webp`, `.gif` и `.svg`. Если статический сервер разрешает просмотр содержимого папки, игра подхватит файл автоматически. Для хостингов без листинга директорий добавьте имя файла в `assets/attributes/manifest.json`.

Например, файл `Military ticket.png` появится в игре как предмет с названием `Military ticket`.

## Автообновление костюмов

Чтобы добавить новый костюм игрока, создайте папку в `assets/player/`, например `assets/player/winter/`. Внутри должны быть пять PNG-спрайтов с фиксированными именами:

- `idle.png`
- `jump.png`
- `run1.png`
- `run2.png`
- `run3.png`

При каждом запуске игра сначала читает `assets/player/manifest.json`, а если файл недоступен — пробует листинг `assets/player/`. Такой manifest надёжнее на хостингах, где directory listing отключён или кэшируется. Имя `id` используется как идентификатор костюма в экспортируемом JSON уровня.

Пример `assets/player/manifest.json`:

```json
{
  "costumes": [
    { "id": "young", "title": "Young", "path": "assets/player/young" },
    { "id": "army", "title": "Army", "path": "assets/player/army" }
  ]
}
```

Если для костюма нужны нестандартные пути к отдельным позам, можно добавить объект `sprites` с ключами `idle`, `jump`, `run1`, `run2`, `run3`.

## Редактор уровня в игре

1. Нажмите `Начать игру`.
2. Нажмите кнопку `Редактор` в верхней панели.
3. Выберите инструмент `Блок`, `Предмет` или `Костюм`.
4. Если выбран `Предмет`, во втором списке выберите конкретную картинку из `assets/attributes/` — например `Laptop` или `Military ticket`.
5. Если выбран `Костюм`, выберите костюм из списка папок `assets/player/`. Клик добавит вертикальный чекпоинт на всю высоту игрового поля: когда игрок дойдёт до него, спрайты игрока сменятся на выбранный костюм.
6. Кликните по canvas, чтобы добавить объект.
7. Используйте `Alt + клик`, `Shift + клик` или правую кнопку мыши, чтобы удалить объект выбранного типа.
8. Нажмите `Экспорт`, чтобы скопировать JSON уровня с блоками, предметами и чекпоинтами костюмов и при необходимости перенести его в `CONFIG.level`.
9. Нажмите `Сброс`, чтобы удалить сохранённый в браузере вариант и вернуться к дефолтному уровню.

Изменения редактора автоматически сохраняются в `localStorage` браузера под ключом `sario.level`.
